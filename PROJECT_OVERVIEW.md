# 📦 AI SALES AGENT BOT — ПОЛНЫЙ СЛЕПОК ПРОЕКТА

## 1. 🎯 Общее описание (TL;DR)

**AI Sales Agent Bot** — личный Telegram-бот для автоматизации холодных B2B-продаж digital-услуг малому бизнесу. Бот находит потенциальных клиентов через OpenStreetMap, анализирует их потенциал с помощью Claude AI, генерирует персонализированные сообщения для первого контакта и управляет простой CRM с напоминаниями прямо в Telegram.

**Проблема:** Ручной поиск клиентов по городам медленный, бессистемный и требует анализа каждой компании вручную. Невозможно отследить, кому уже написали, кто ответил, когда связаться повторно.

**Ценность:** Один человек за 5 минут получает список из 60 профилированных компаний с готовыми сообщениями и CRM для отслеживания статусов — работа, на которую вручную уходит день.

---

## 2. 👥 Пользователи и роли

| Роль | Возможности | Ограничения |
|------|------------|-------------|
| **Владелец бота** (single user) | • Поиск компаний по городу и категории<br>• AI-анализ компаний (оценка 0-100, слабые места, наличие онлайн-записи)<br>• Генерация холодных сообщений (2 варианта)<br>• Управление лидами (статусы, заметки, фильтры)<br>• Напоминания (быстрые и кастомные даты)<br>• Настройка Chat Monitor (пассивный сбор лидов из Telegram-чатов)<br>• Экспорт лидов в CSV | • Видит только свои данные<br>• Лимит AI-вызовов не настраивается через UI<br>• Не может делегировать доступ<br>• Часовой пояс напоминаний — только UTC |

**Примечание:** Бот спроектирован как личный инструмент. Архитектурно поддерживает multi-user (data scoping по `owner_tg_id`), но UI не рассчитан на командную работу.

## 3. 🗺️ Пользовательские сценарии (User Flows)

### 3.1. Первый запуск и поиск компании

**Шаг 1:** Пользователь отправляет `/start` в Telegram  
→ **Под капотом:** Бот проверяет наличие пользователя в таблице `users`. Если нет — создаёт запись с `tg_user_id` и `username`. Очищает FSM-состояние (если было).  
→ **Видит:** Приветственное сообщение + главное меню с 4 кнопками: «🔍 Новый поиск», «📊 Мои лиды», «📡 Chat Monitor», «⚙️ Настройки».

**Шаг 2:** Нажимает «🔍 Новый поиск»  
→ **Под капотом:** FSM переходит в состояние `SearchFSM.waiting_city`. Сохраняет состояние диалога в Redis-like хранилище aiogram.  
→ **Видит:** «В каком городе ищем компании? Напиши название (например, Астана, Алматы, Москва)».

**Шаг 3:** Вводит город текстом, например: `Астана`  
→ **Под капотом:** Хендлер валидирует длину (не более 100 символов). Санитизирует ввод (удаляет `\`, `"`, `[`, `]`, `~` для защиты от QL-инъекций). FSM переходит в `SearchFSM.waiting_category`. Сохраняет город в `FSMContext.data`.  
→ **Видит:** «Выбери категорию:» + 8 кнопок (Барбершоп, Салон красоты, Кафе, Ресторан, Фитнес-клуб, Стоматология, Автосервис, Магазин цветов).

**Шаг 4:** Нажимает кнопку категории, например: «☕ Кафе»  
→ **Под капотом:**  
  1. Формирует Overpass QL-запрос: `node["amenity"="cafe"](area:"Астана"); out 60;`  
  2. Отправляет POST-запрос к `https://overpass-api.de/api/interpreter` через aiohttp (таймаут 30 сек)  
  3. Парсит JSON-ответ, извлекает: `name`, `addr:*`, `phone`, `website`, `contact:*`  
  4. Если найдено 0 компаний → сообщение «Ничего не нашлось»  
  5. Если найдено N компаний → сохраняет результаты в `FSMContext.data['results']`, FSM переходит в `SearchFSM.browsing`  
  6. Кэширует результаты в памяти (TTL 30 минут) по ключу `(город, категория)` для ускорения повторных запросов  
→ **Видит:** «Нашёл 42 компании. Листай карточки:» + карточка первой компании:
```
📍 Кофейня «Сartel»
Адрес: ул. Достык, 12
📞 +7 777 123 4567
🌐 cartel.kz

🔗 2ГИС | 📸 Instagram

[Сохранить в лиды] [🤖 Анализ]
[◀ 1/42 ▶]
```

**Шаг 5:** Нажимает «🤖 Анализ»  
→ **Под капотом:**  
  1. **Автосохранение:** Проверяет через `find_lead_by_name_address`, есть ли уже лид с таким именем+адресом. Если нет — создаёт запись в таблице `leads` со статусом `new`, source `osm`.  
  2. **Проверка лимита:** Считает число LLM-вызовов за последние 24 часа из таблицы `llm_call_log`. Если >= дневного лимита (по умолчанию не задан) → останавливается с сообщением «Достигнут дневной лимит».  
  3. **Idempotency:** Проверяет in-memory set `_analysis_in_progress[(owner_tg_id, lead_id)]`. Если анализ уже идёт → отказывает с alert «Анализ уже выполняется».  
  4. **LLM-запрос:** Формирует промпт:
     - Системная инструкция: «Ты — эксперт по анализу бизнеса. Оцени потенциал компании для продажи услуг digital-маркетинга / онлайн-записи.»
     - User message: `Компания: Cartel\nАдрес: ул. Достык, 12\nТелефон: +7 777 123 4567\nСайт: cartel.kz\n\nОцени 0-100, опиши слабые места, проверь наличие онлайн-записи.`
  5. Отправляет через выбранный LLM-провайдер (Anthropic Claude или OpenAI-совместимый, например OpenRouter). Таймаут 60 сек.  
  6. Парсит JSON-ответ: `{"score": 75, "analysis": "...", "has_online_booking": false}`  
  7. Сохраняет в БД: `UPDATE leads SET ai_score=75, ai_analysis='...', has_online_booking=False WHERE id=X`  
  8. Пишет запись в `llm_call_log` с timestamp.  
  9. Освобождает idempotency-слот.  
→ **Видит:** «🔄 Анализирую компанию…» (5-15 секунд) → Обновлённая карточка лида:
```
📊 Лид #42: Кофейня «Cartel»

📍 Адрес: ул. Достык, 12
📞 +7 777 123 4567
🌐 cartel.kz

⭐ AI-оценка: 75/100
📊 Онлайн-запись: ❌ Нет

🔥 Насколько нужен мой сервис: ВЫСОКИЙ интерес
(нет онлайн-записи + средняя оценка)

💡 AI-анализ:
У компании хороший сайт, но отсутствует система онлайн-бронирования столиков. Клиенты должны звонить. Слабое место — нет интеграции с соцсетями для записи. Рекомендую предложить Telegram-бота для бронирования.

Статус: 🆕 Новый

[Статус] [📝 Заметка] [🔄 Анализ] [💬 Сообщения]
[⏰ Напомнить] [↩ К лидам]
```

**Шаг 6:** Нажимает «💬 Сообщения для контакта»  
→ **Под капотом:**  
  1. Проверяет наличие `ai_analysis`. Если нет → alert «Сначала сделай AI-анализ».  
  2. Формирует промпт:
     ```

     1) Короткое (2-3 предложения)
     ```

  3. Отправляет LLM-запрос, парсит JSON-ответ: `{"short": "...", "long": "..."}`  
  4. Сохраняет в `FSMContext.data['generated_messages']` для возможности редактирования.  
→ **Видит:**
```

<pre>
```

**Шаг 8:** Возвращается в бота, нажимает «↩ К карточке» → «Статус» → выбирает «✍️ Написали»  
→ **Под капотом:** `UPDATE leads SET status='written', updated_at=NOW() WHERE id=42`

### 3.2. Напоминание срабатывает (автоматически, через 3 дня)

**Фоновый процесс** (каждые 60 секунд):  
1. Выбирает из БД: `SELECT * FROM reminders WHERE remind_at <= NOW() AND is_sent=False`  
2. Для каждого напоминания:
   - **ДО отправки** помечает: `UPDATE reminders SET is_sent=True WHERE id=X` (защита от дублей при рестарте)  
   - Загружает лид: `SELECT name FROM leads WHERE id=lead_id`  
   - Формирует текст: `Напоминание по лиду <b>Кофейня «Cartel»</b>\n\nПора связаться: уже прошло 3 дня после первого сообщения.`  
   - Отправляет через `bot.send_message(owner_tg_id, text)` с тремя уровнями fallback (HTML+эмодзи → HTML без эмодзи → plain text)  
   - Если отправка упала (ошибка сети/Telegram API) → откатывает: `UPDATE reminders SET is_sent=False WHERE id=X` — повторит на следующей итерации  

→ **Пользователь видит** (через 3 дня в 16:30 UTC):
```
⏰ Напоминание по лиду Кофейня «Cartel»

Пора связаться: уже прошло 3 дня после первого сообщения.
```

### 3.5. Chat Lead Monitor — пассивный сбор лидов

**Настройка (один раз):**

**Шаг 1:** Пользователь заполняет `.env`:
```env
CHAT_MONITOR_OWNER_TG_ID=123456789
CHAT_MONITOR_API_ID=12345678
CHAT_MONITOR_API_HASH=abcdef1234567890
CHAT_MONITOR_PHONE=+77001234567
CHAT_MONITOR_CHATS=@open_nails_chat,-1001234567890
CHAT_MONITOR_MIN_SCORE=0.7
```

**Шаг 2:** Запускает отдельный процесс: `python -m chat_monitor.runner`  
→ **Под капотом:**  
  1. Проверяет наличие всех настроек Telethon (`_ensure_ready`). Если что-то пусто → краш с понятным сообщением.  
  2. Создаёт `TelegramClient(session_path, api_id, api_hash)`.  
  3. При первом запуске Telethon запрашивает код авторизации → пользователь вводит код из Telegram (не SMS).  
  4. Создаёт файл сессии `chat_monitor.session` (чувствительный, в `.gitignore`).  
  5. Регистрирует обработчик `@client.on(events.NewMessage())`.  
  6. Запускает бесконечный цикл: `client.run_until_disconnected()`.

**Работа в фоне:**

При каждом новом сообщении в любом чате:  
1. **Загружает runtime-конфиг из БД** (каждый раз заново — настройки могут измениться через бота):
   ```sql
   SELECT is_enabled, chats, min_score FROM chat_monitor_settings WHERE owner_tg_id=X
   ```
2. Проверяет `is_enabled=True`. Если False → пропускает.  
3. Проверяет, входит ли `event.chat_id` или `@username` чата в список `chats`. Если нет → пропускает.  
4. Извлекает данные кандидата:
   - `text` = `event.message.message` (до 2000 символов обрезается для LLM)
   - `sender_id` = `event.sender_id`
   - `sender_username` = `@username` автора (если есть)
   - `chat_title` = название чата
   - `message_id` = id сообщения
   - `message_date` = timestamp
5. **Keyword-фильтр** (без сети, быстрый):
   - Проверяет наличие ключевых слов nail-ниши: `маникюр`, `педикюр`, `ногти`, `мастер`, `салон`, `покрытие`, `гель-лак`, `френч`, `дизайн`, `запись` и т.д.  
   - Регистронезависимый поиск через regex.  
   - Если **0 совпадений** → пропускает (экономит LLM-вызовы).  
6. **LLM-скоринг**:
   - Формирует промпт:
     ```
     Оцени сообщение из Telegram-чата на предмет релевантности для поиска nail-мастеров:
     
     Сообщение: "{text}"
     
     Критерии:
     - Автор ищет мастера маникюра/педикюра?
     - Или мастер предлагает свои услуги?
     - Или обсуждение качества работы мастера?
     
     Ответь JSON: {"score": 0.0-1.0, "reasoning": "...", "is_solo_master": true/false}
     ```
   - Отправляет LLM-запрос (таймаут 60 сек).  
   - Парсит ответ: `{"score": 0.85, "reasoning": "Автор ищет мастера на дом", "is_solo_master": true}`  
7. Если `score < min_score` (например, 0.7) → пропускает (логирует).  
8. **Дедупликация:**
   ```sql
   SELECT id FROM leads 
   WHERE owner_tg_id=X AND chat_user_id=sender_id AND chat_message_id=message_id
   ```
   Если найдено → пропускает (уже сохранён).  
9. **Сохранение как лид:**
   ```sql
   INSERT INTO leads (
     owner_tg_id, name, source, niche,
     source_chat, chat_username, chat_user_id, chat_message_id,
     message_text, message_date, relevance_score, status

     X, '@username', 'chat_monitor', 'nail',

   ```
10. **Уведомление владельцу:**
    ```
    🔔 Новый лид из Chat Monitor!
    
    📡 Чат: Открытые ногти Астаны
    👤 Автор: @ivanova
    📊 Релевантность: 85%

    "Ищу мастера маникюра на дом, район Есиль. Кто может завтра?"

    ```

→ **Пользователь видит** уведомление в боте, открывает карточку лида, видит:
